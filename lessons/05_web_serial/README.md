## 이번 단계에서 만들 것

USB 케이블로 연결한 Arduino UNO의 센서값을 웹페이지에서 읽습니다. 연결 버튼, 센서값 카드, 수신 원문, 오류 안내가 있는 통신 확인 페이지를 완성합니다.

키트가 없을 때는 가상 센서 모드로 같은 JSON 해석 과정을 시험할 수 있습니다.

## 학습목표

- 시리얼 통신의 속도와 데이터 구분 규칙을 설명할 수 있다.
- Arduino가 보낸 한 줄의 JSON을 웹에서 센서값으로 나눌 수 있다.
- 웹에서 Arduino로 명령을 보내고 응답을 확인할 수 있다.
- 연결·형식·권한 오류를 구분해 해결할 수 있다.

## 핵심 질문

연속해서 도착하는 시리얼 데이터를 웹이 정확히 한 묶음씩 읽게 하려면 어떤 규칙이 필요할까요?

## 로컬 웹서버 실행

프로젝트 루트에서 다음 명령을 실행합니다.

```bash
python3 -m http.server 8000
```

브라우저에서 다음 주소를 엽니다.

```text
http://localhost:8000/web/05_web_serial/
```

`localhost`는 인터넷 서버가 아니라 지금 사용 중인 컴퓨터를 뜻합니다. Python의 간단한 정적 파일 서버가 현재 폴더의 HTML·CSS·JavaScript를 브라우저에 전달합니다.

## 통신 규칙

Arduino와 웹은 `115200 baud`로 통신합니다. 데이터 하나는 JSON 객체 하나로 만들고 줄바꿈으로 끝냅니다. 이 형식을 JSON Lines라고 합니다.

```json
{"type":"sensor","pot":512,"light":640,"touch":false,"distance":31.4,"brightness":40,"color":1}
```

| 이름 | 뜻 | 값의 예 |
|---|---|---|
| `type` | 데이터 종류 | `sensor` |
| `pot` | 가변저항값 | 0~1023 |
| `light` | 조도센서값 | 0~1023 |
| `touch` | 터치 감지 | `true`, `false` |
| `distance` | 거리(cm) | 31.4, 측정 실패 시 `null` |
| `brightness` | 현재 LED 밝기 | 0~80 |
| `color` | 현재 색 번호 | 0~3 |

웹에서 Arduino로 보내는 명령도 줄바꿈으로 끝냅니다.

| 명령 | 응답 |
|---|---|
| `PING` | 연결 확인용 `pong` 응답 |
| `GET` | 현재 센서값 한 줄 즉시 전송 |
| 그 밖의 명령 | `unknown_command` 오류 |

### 왜 줄바꿈이 필요한가?

USB 시리얼 데이터는 파일처럼 한 번에 도착하지 않을 수 있습니다. JSON 한 줄이 여러 조각으로 나뉘거나, 여러 줄이 붙어서 도착할 수도 있습니다. 웹은 도착한 글자를 버퍼에 모은 뒤 `\n`을 발견했을 때 한 데이터가 끝났다고 판단합니다.

```text
도착한 바이트 → 글자로 변환 → 버퍼에 저장 → 줄바꿈에서 분리 → JSON 해석
```

따라서 Arduino의 `Serial.println()`과 웹의 줄 단위 처리가 같은 규칙을 사용해야 합니다.

## Part 4 코드에 통신 기능 쌓기

### 1. 사람이 읽는 출력을 JSON으로 바꾸기

Part 4의 자신의 Arduino 파일을 엽니다. 새 완성 파일을 업로드하지 않습니다. 먼저 `sendSensorData()`를 작성하고 기존 `printSystemState()` 호출을 이 함수 호출로 바꿉니다.

처음에는 가변저항 하나만 JSON으로 보내 형식을 확인합니다.

```json
{"type":"sensor","pot":512}
```

정상 출력이 확인되면 조도, 터치, 거리, 밝기 항목을 한 개씩 추가합니다. 항목을 추가할 때마다 쉼표와 따옴표를 확인하고 다시 실행합니다.

### 2. 전송 주기 함수 추가

`sendSensorDataAtInterval()`을 작성하고 `loop()`의 마지막에 호출합니다. 전송 간격을 200ms와 500ms로 바꾸어 센서 읽기와 통신이 동시에 계속되는지 비교합니다.

### 3. 웹 연결 뼈대에서 기능 쌓기

`web/05_web_serial/starter/index.html`을 GitHub Pages 또는 로컬 서버로 엽니다. `Arduino 연결`을 누르고 수신 원문이 보이는지 확인합니다.

원문이 보이면 `handleLine()`에 `JSON.parse()`를 추가하고, 센서 이름과 값을 하나씩 화면에 표시합니다.

### 4. 웹 명령 수신 함수 추가

자신의 Arduino 코드에 `readWebCommand()`를 추가하고 `loop()`에 호출합니다. 먼저 `PING`에 `pong`을 응답하게 한 뒤 `GET`을 추가합니다. 시리얼 모니터와 웹페이지는 같은 포트를 동시에 사용할 수 없으므로 웹 연결 전에는 시리얼 모니터를 닫습니다.

### 5. 누적 결과 확인

자신이 만든 웹페이지를 연결한 뒤 다음을 확인합니다.

- 가변저항을 돌리면 `가변저항`과 `밝기` 값이 변하는가?
- 조도센서를 가리면 `조도` 값이 변하는가?
- 터치센서를 누르면 `터치` 상태가 변하는가?
- 초음파센서 앞 물체를 움직이면 `거리`가 변하는가?
- `PING`을 누르면 `pong` 응답이 오는가?

### 6. 키트 없이 가상 센서 시험

`가상 센서 시작`을 누릅니다. 웹페이지가 만든 가상 JSON도 실제 수신 데이터와 같은 `handleLine()` 함수를 거칩니다. 센서 카드와 원문 로그가 바뀌면 웹의 데이터 처리 부분은 먼저 확인한 것입니다.

가상 모드 성공은 Arduino 연결 성공을 뜻하지 않습니다. 실제 포트 연결은 키트로 다시 확인해야 합니다.

## 코드가 하는 일

### Arduino

1. 센서값을 읽습니다.
2. 200 ms마다 JSON 한 줄을 만듭니다.
3. `Serial.print()`로 웹에 보냅니다.
4. 웹이 보낸 글자를 줄바꿈까지 모읍니다.
5. `PING`과 `GET`을 구분해 응답합니다.

### 웹

1. 사용자가 선택한 시리얼 포트를 엽니다.
2. 도착한 바이트를 글자로 바꿉니다.
3. 줄바꿈을 기준으로 한 줄씩 나눕니다.
4. `JSON.parse()`로 이름과 값을 구분합니다.
5. 카드와 로그에 결과를 표시합니다.

## 2차시 활동. 통신 오류를 일부러 만들고 복구하기

1. 정상 상태에서 `PING`을 보내 `pong`을 확인합니다.
2. Arduino IDE의 시리얼 모니터를 열어 같은 포트를 점유해 봅니다.
3. 웹 연결이 되지 않는 것을 확인한 뒤 시리얼 모니터를 닫습니다.
4. 웹에서 다시 연결합니다.
5. 가상 센서를 켜고 잘못된 JSON 예시를 입력해 오류 표시를 관찰합니다.

오류를 만든 뒤 원래 상태로 복구할 수 있어야 실습을 완료한 것입니다.

| 문제 | 원인 | 해결 방법 |
|---|---|---|

| 거리가 `측정 실패`로 표시됨 | 초음파 반사 신호 없음 | 물체 방향과 거리를 바꾸고 결선 확인 |

## 도전과제

1. 전송 간격을 200 ms에서 100 ms와 500 ms로 바꾸고 화면 변화와 로그 읽기 편의성을 비교합니다.
2. `GET` 버튼을 웹에 추가해 원하는 순간의 센서값만 받습니다.

4. 데이터에 `uptime`을 추가해 Arduino가 켜진 시간을 웹에 표시합니다.

## 실제 키트에서 확인한 값

- 터치센서는 접촉 중 HIGH
- 조도센서는 밝을 때 약 15, 가렸을 때 약 850으로 어두울수록 값이 증가
- 초음파센서는 너무 가까운 물체를 제외하고 10cm·30cm 시험에서 실제 거리와 대략 일치
- 네오픽셀 밝기 상한은 80
- 센서 JSON 전송 간격은 200ms
