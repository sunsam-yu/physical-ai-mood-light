# 5단계. 아두이노와 웹 연결

## 이번 단계에서 만들 것

USB 케이블로 연결한 Arduino UNO의 센서값을 웹페이지에서 읽습니다. 연결 버튼, 센서값 카드, 수신 원문, 오류 안내가 있는 통신 확인 페이지를 완성합니다.

키트가 없을 때는 가상 센서 모드로 같은 JSON 해석 과정을 시험할 수 있습니다.

## 학습목표

- 시리얼 통신의 속도와 데이터 구분 규칙을 설명할 수 있다.
- Arduino가 보낸 한 줄의 JSON을 웹에서 센서값으로 나눌 수 있다.
- 웹에서 Arduino로 명령을 보내고 응답을 확인할 수 있다.
- 연결·형식·권한 오류를 구분해 해결할 수 있다.

## 핵심 질문

연속해서 도착하는 시리얼 데이터를 웹이 정확히 한 묶음씩 읽게 하려면 어떤 규칙이 필요할까요?

## 준비물

- Arduino UNO와 USB 케이블
- 2단계에서 점검한 센서와 네오픽셀
- Arduino IDE
- Chrome 또는 Edge
- Arduino 코드: [`arduino/05_web_serial`](../../arduino/05_web_serial)
- 웹 코드: [`web/05_web_serial`](../../web/05_web_serial)

## 통신 규칙

Arduino와 웹은 `115200 baud`로 통신합니다. 데이터 하나는 JSON 객체 하나로 만들고 줄바꿈으로 끝냅니다. 이 형식을 JSON Lines라고 합니다.

```json
{"type":"sensor","pot":512,"light":640,"touch":false,"distance":31.4,"brightness":40,"color":1}
```

| 이름 | 뜻 | 값의 예 |
|---|---|---|
| `type` | 데이터 종류 | `sensor` |
| `pot` | 가변저항값 | 0~1023 |
| `light` | 조도센서값 | 0~1023 |
| `touch` | 터치 감지 | `true`, `false` |
| `distance` | 거리(cm) | 31.4, 측정 실패 시 `null` |
| `brightness` | 현재 LED 밝기 | 0~80 |
| `color` | 현재 색 번호 | 0~3 |

웹에서 Arduino로 보내는 명령도 줄바꿈으로 끝냅니다.

| 명령 | 응답 |
|---|---|
| `PING` | 연결 확인용 `pong` 응답 |
| `GET` | 현재 센서값 한 줄 즉시 전송 |
| 그 밖의 명령 | `unknown_command` 오류 |

## 따라 하기

### 1. 시작 코드로 데이터 형식 확인

`arduino/05_web_serial/01_start/01_start.ino`를 엽니다. 보드와 포트를 선택하고 업로드합니다.

시리얼 모니터를 `115200 baud`로 열어 가변저항값이 한 줄씩 출력되는지 확인합니다.

```json
{"type":"sensor","pot":512}
```

확인이 끝나면 시리얼 모니터를 반드시 닫습니다. 시리얼 모니터와 웹페이지는 같은 포트를 동시에 사용할 수 없습니다.

### 2. 학생용 시작 웹 열기

`web/05_web_serial/starter/index.html`을 GitHub Pages 또는 로컬 서버로 엽니다. 파일을 더블클릭한 주소보다 HTTPS 또는 localhost 실행을 권장합니다.

`Arduino 연결`을 누르고 목록에서 Arduino 포트를 선택합니다. 데이터 원문이 한 줄씩 보이는지 확인합니다.

### 3. 완성 Arduino 코드 업로드

`arduino/05_web_serial/02_complete/02_complete.ino`를 업로드합니다. 센서 네 종류와 현재 조명 상태가 JSON 한 줄에 함께 출력됩니다.

### 4. 완성 웹페이지 열기

`web/05_web_serial/index.html`을 엽니다. 연결한 뒤 다음을 확인합니다.

- 가변저항을 돌리면 `가변저항`과 `밝기` 값이 변하는가?
- 조도센서를 가리면 `조도` 값이 변하는가?
- 터치센서를 누르면 `터치` 상태가 변하는가?
- 초음파센서 앞 물체를 움직이면 `거리`가 변하는가?
- `PING`을 누르면 `pong` 응답이 오는가?

### 5. 키트 없이 가상 센서 시험

`가상 센서 시작`을 누릅니다. 웹페이지가 만든 가상 JSON도 실제 수신 데이터와 같은 `handleLine()` 함수를 거칩니다. 센서 카드와 원문 로그가 바뀌면 웹의 데이터 처리 부분은 먼저 확인한 것입니다.

가상 모드 성공은 Arduino 연결 성공을 뜻하지 않습니다. 실제 포트 연결은 키트로 다시 확인해야 합니다.

## 코드가 하는 일

### Arduino

1. 센서값을 읽습니다.
2. 200 ms마다 JSON 한 줄을 만듭니다.
3. `Serial.print()`로 웹에 보냅니다.
4. 웹이 보낸 글자를 줄바꿈까지 모읍니다.
5. `PING`과 `GET`을 구분해 응답합니다.

### 웹

1. 사용자가 선택한 시리얼 포트를 엽니다.
2. 도착한 바이트를 글자로 바꿉니다.
3. 줄바꿈을 기준으로 한 줄씩 나눕니다.
4. `JSON.parse()`로 이름과 값을 구분합니다.
5. 카드와 로그에 결과를 표시합니다.

## 자주 생기는 오류

| 문제 | 원인 | 해결 방법 |
|---|---|---|
| 연결 버튼이 비활성화됨 | 지원하지 않는 브라우저 | Chrome 또는 Edge 사용 |
| 포트 목록에 Arduino가 없음 | 케이블·드라이버·포트 문제 | 데이터 전송 USB 케이블인지 확인하고 다시 연결 |
| 포트를 열 수 없음 | Arduino IDE 시리얼 모니터가 사용 중 | 시리얼 모니터와 플로터 닫기 |
| 연결 직후 잠시 값이 없음 | 포트를 열며 UNO가 재시작됨 | 2~3초 기다리기 |
| JSON 형식 오류 | 설명 문장과 JSON이 섞임 | Arduino 출력은 JSON만 한 줄씩 보내기 |
| 거리가 `측정 실패`로 표시됨 | 초음파 반사 신호 없음 | 물체 방향과 거리를 바꾸고 결선 확인 |
| 가상 센서는 되지만 실제 연결은 안 됨 | 웹 처리와 하드웨어 연결은 별도 | 케이블·포트·속도·업로드 코드를 순서대로 점검 |

## 도전과제

1. 전송 간격을 200 ms에서 100 ms와 500 ms로 바꾸고 화면 변화와 로그 읽기 편의성을 비교합니다.
2. `GET` 버튼을 웹에 추가해 원하는 순간의 센서값만 받습니다.
3. 일부러 JSON의 쉼표 하나를 지워 형식 오류가 어떻게 표시되는지 확인합니다.
4. 데이터에 `uptime`을 추가해 Arduino가 켜진 시간을 웹에 표시합니다.

## 키트 검증 후 확정할 값

- 실제 터치센서가 눌렸을 때 HIGH인지 LOW인지
- 조도센서값이 밝을 때 증가하는지 감소하는지
- 초음파센서의 교실 환경 측정 범위와 오차
- 네오픽셀의 안전하고 보기 좋은 최대 밝기
