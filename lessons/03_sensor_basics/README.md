# 3단계. 센서별 기본 실습

[이전 단계: 키트 조립 및 부품 점검](../02_component_test/README.md) · [프로젝트 전체 보기](../../README.md) · [다음 단계: 모듈형 코드](../04_modular_controller/README.md)

## 이번 단계에서 만들 것

가변저항, 조도센서, 터치센서, 초음파센서, 네오픽셀을 하나씩 실행합니다. 센서의 물리적 변화가 어떤 숫자로 바뀌는지 확인하고, 나중에 자동 조명의 기준으로 사용할 실제 값을 기록합니다.

## 학습목표

- 아날로그 입력과 디지털 입력의 차이를 설명할 수 있다.
- 초음파의 왕복 시간으로 거리를 계산할 수 있다.
- RGB 세 값으로 네오픽셀 색을 만들 수 있다.
- 센서값의 최솟값·평상시 값·최댓값을 측정할 수 있다.

## 핵심 질문

센서 주변의 물리적 변화는 아두이노에서 어떤 숫자로 표현될까?

## 준비물

- Arduino UNO와 USB 케이블
- 가변저항, 조도센서, 터치센서, 초음파센서
- 네오픽셀 8구
- Arduino IDE
- `Adafruit NeoPixel` 라이브러리

## 핀 배치

| 부품 | Arduino UNO 핀 |
|---|---|
| 가변저항 | A0 |
| 조도센서 | A5 |
| 터치센서 | D13 |
| 초음파센서 TRIG | D2 |
| 초음파센서 ECHO | D3 |
| 네오픽셀 | D7 |

키트의 실제 커넥터 배열과 결선 사진은 실물 확인 후 추가합니다. 전원을 연결하기 전에 `5V·GND·신호선`의 방향을 반드시 확인합니다.

## 하나의 코드에 입력 함수를 쌓는 순서

Part 2에서 저장한 `physical_ai_mood_light.ino`를 엽니다. 아래 독립 예제는 정답을 확인할 때만 사용하고, 실습은 자신의 파일에 함수를 하나씩 직접 타이핑하며 진행합니다.

1. 전역 변수 `potValue`를 선언하고 `readPotentiometer()`를 작성합니다.
2. `loop()`에 `readPotentiometer();`를 추가한 뒤 값이 변하는지 확인합니다.
3. 같은 방법으로 `readLightSensor()`를 작성하고 호출문을 한 줄 더 추가합니다.
4. `readTouchSensor()`를 추가해 HIGH와 LOW를 확인합니다.
5. `readDistanceSensor()`를 추가해 초음파 왕복 시간을 거리로 바꿉니다.
6. 마지막으로 센서값을 한 줄에 출력하는 `printSensorValues()`를 작성합니다.

함수를 여러 개 먼저 복사한 뒤 한꺼번에 실행하지 않습니다. 하나를 작성하고 호출하고 시험한 뒤 다음 함수로 넘어갑니다.

```cpp
void loop() {
  readPotentiometer();
  readLightSensor();
  readTouchSensor();
  readDistanceSensor();
  printSensorValues();
}
```

## 실습 1. 가변저항

자신의 코드에 `readPotentiometer()`를 작성하고 손잡이를 끝에서 끝까지 천천히 돌립니다. [가변저항 참고 코드](../../arduino/03_sensor_basics/01_potentiometer/01_potentiometer.ino)는 실행과 기록을 마친 뒤 비교합니다.

- `analogRead()`는 A0의 전압을 `0~1023` 정수로 바꿉니다.
- `map()`은 이 값을 보기 쉬운 `0~100%`로 바꿉니다.
- 손잡이를 같은 위치로 되돌렸을 때 비슷한 값이 나오는지 확인합니다.

|---|---|---|

| 가운데 |  |  |
| 오른쪽 끝 |  |  |

## 실습 2. 조도센서

같은 파일에 `readLightSensor()`와 호출문을 추가합니다. 두 입력값이 함께 갱신되는 것을 확인한 뒤 센서를 가리거나 빛을 비춥니다. [조도센서 참고 코드](../../arduino/03_sensor_basics/02_light_sensor/02_light_sensor.ino)는 마지막에 비교합니다.

조도센서는 키트의 회로 구성에 따라 밝을 때 값이 커질 수도, 작아질 수도 있습니다. 숫자의 크기를 외우지 말고 실제 변화 방향을 확인합니다.

| 주변 조건 | 측정값 |
|---|---|
| 손으로 완전히 가림 |  |
| 평상시 실내 |  |
| 가까이에서 빛을 비춤 |  |

측정 결과: 밝아질수록 값이 `커진다 / 작아진다`.

## 실습 3. 터치센서

같은 파일에 `readTouchSensor()`와 호출문을 추가합니다. [터치센서 참고 코드](../../arduino/03_sensor_basics/03_touch_sensor/03_touch_sensor.ino)는 마지막에 비교합니다.

- 디지털 입력은 연속된 숫자 대신 `HIGH` 또는 `LOW` 두 상태를 읽습니다.
- 예제에서는 `HIGH`를 `감지`, `LOW`를 `없음`으로 표시합니다.
- 한 번 누를 때 여러 번 감지되는지, 손을 떼면 바로 원래 상태로 돌아오는지 확인합니다.

| 상태 | 시리얼 모니터 표시 |
|---|---|
| 누르지 않음 |  |
| 누름 |  |

실제 키트에서 출력이 반대로 나타나면 4단계 코드의 판정 조건을 수정합니다.

## 실습 4. 초음파센서

같은 파일에 `readDistanceSensor()`와 호출문을 추가하고 평평한 물체를 센서 정면에 둡니다. [초음파센서 참고 코드](../../arduino/03_sensor_basics/04_ultrasonic_sensor/04_ultrasonic_sensor.ino)는 마지막에 비교합니다.

아두이노는 초음파가 물체에 반사되어 돌아오는 시간을 마이크로초 단위로 측정합니다.

`거리(cm) = 왕복 시간(μs) × 0.0343 ÷ 2`

마지막에 2로 나누는 이유는 측정한 시간이 센서에서 물체까지 갔다가 돌아온 왕복 시간이기 때문입니다.

| 자로 잰 거리 | 센서 측정값 | 차이 |
|---|---|---|
| 10 cm |  |  |
| 30 cm |  |  |
| 100 cm |  |  |

`측정 실패`는 제한 시간 안에 반사파가 돌아오지 않았다는 뜻입니다. 센서 앞 물체의 각도와 TRIG·ECHO 연결을 확인합니다.

## 실습 5. 네오픽셀

Part 2에서 만든 `setupNeoPixel()` 아래에 시험용 출력 함수를 잠시 작성합니다. 색을 확인한 뒤 함수는 남겨 두되 `loop()`의 시험 호출은 제거합니다. [네오픽셀 참고 코드](../../arduino/03_sensor_basics/05_neopixel/05_neopixel.ino)는 마지막에 비교합니다.

- RGB는 빨강, 초록, 파랑의 밝기를 각각 `0~255`로 정합니다.
- `(255, 0, 0)`은 빨강, `(0, 0, 255)`는 파랑입니다.
- `BRIGHTNESS`는 처음에 `50`을 유지합니다. USB 전원의 부담을 줄이기 위해 최대 밝기로 시험하지 않습니다.

| 만들 색 | R | G | B | 실제 색 확인 |
|---|---|---|---|---|
| 빨강 | 255 | 0 | 0 |  |
| 초록 | 0 | 255 | 0 |  |
| 파랑 | 0 | 0 | 255 |  |
| 내가 만든 색 |  |  |  |  |

## 정상 작동 확인표

- [ ] 가변저항의 양 끝과 가운데 값이 구분된다.
- [ ] 조도센서값의 변화 방향을 확인했다.
- [ ] 터치센서의 평상시·감지 상태를 확인했다.
- [ ] 10·30·100 cm에서 초음파 거리를 측정했다.
- [ ] 네오픽셀 8개가 같은 색으로 켜지고 색이 순서대로 바뀐다.
- [ ] 측정값을 기록표에 입력했다.

## 자주 생기는 오류

| 증상 | 먼저 확인할 것 |
|---|---|
| 시리얼 모니터에 값이 안 나옴 | 포트 선택, `115200 baud`, 업로드 완료 여부 |
| 값이 계속 0 또는 1023 | 센서의 5V·GND·신호선 연결 |
| 터치 상태가 반대로 나옴 | 실제 키트의 평상시 출력이 HIGH인지 확인 |
| 거리가 크게 흔들림 | 물체가 센서 정면에 있는지, 표면이 평평한지 확인 |
| 거리가 계속 측정 실패 | D2·D3와 TRIG·ECHO 연결 확인 |
| 네오픽셀이 켜지지 않음 | D7, 전원 방향, 라이브러리 설치 확인 |

## 도전과제

1. 가변저항 코드의 출력 주기를 `100 ms`와 `1000 ms`로 바꾸어 차이를 비교합니다.
2. 네오픽셀 코드의 RGB 표에 원하는 색을 한 가지 추가합니다.
3. 초음파센서값이 실제 자의 값과 얼마나 차이 나는지 백분율로 계산합니다.
