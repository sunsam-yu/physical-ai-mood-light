# 2단계. 키트 조립 및 부품 점검

[이전 단계: 프로젝트 이해](../01_project_overview/README.md) · [프로젝트 전체 보기](../../README.md)

## 학습목표

- 키트의 센서와 네오픽셀 핀을 확인할 수 있다.
- 상수, 객체, `setup()`, `loop()`의 역할을 구분할 수 있다.
- 핀 설정과 네오픽셀 초기화를 각각 함수로 작성할 수 있다.

## 완성 결과

Part 10까지 이어서 사용할 하나의 Arduino 파일을 만들고, 핀 설정과 네오픽셀 초기화까지 완료합니다. 시험 색 한 가지로 결선과 라이브러리를 확인합니다.

## 준비물

- Arduino UNO와 USB 케이블
- 키트에 포함된 센서 보드와 네오픽셀 8구
- Arduino IDE가 설치된 컴퓨터
- `Adafruit NeoPixel` 라이브러리

## 핀 확인

| 부품 | Arduino UNO 핀 |
|---|---|
| 가변저항 | A0 |
| 조도센서 | A5 |
| 터치센서 | D13 |
| 초음파센서 TRIG | D2 |
| 초음파센서 ECHO | D3 |
| 네오픽셀 | D7 |

전원을 연결하기 전에 매뉴얼의 결선과 위 핀 번호가 일치하는지 확인합니다.

## 내 프로젝트 파일 만들기

이 단계부터 새 예제 파일을 번갈아 열지 않습니다. Arduino IDE에서 `physical_ai_mood_light.ino`를 만들고 Part 10까지 같은 파일을 이어서 사용합니다.

1. 핀 번호와 네오픽셀 객체를 코드 위쪽에 직접 입력합니다.
2. `setupPins()`를 만들고 터치·초음파 핀의 입출력을 설정합니다.
3. `setupNeoPixel()`을 만들고 네오픽셀을 꺼진 상태로 초기화합니다.
4. `setup()`에 두 함수의 호출문을 입력합니다.
5. 업로드한 뒤 오류가 없는지 확인합니다.
6. 네오픽셀에 한 가지 색을 잠시 표시해 결선과 라이브러리만 점검한 뒤 시험 호출은 지웁니다.

```cpp
void setup() {
  Serial.begin(115200);
  setupPins();
  setupNeoPixel();
}

void loop() {
  // Part 3부터 입력 함수가 한 줄씩 늘어납니다.
}
```

이 단계의 목적은 모든 센서를 한꺼번에 작동시키는 것이 아니라, 다음 단계에서 기능을 붙일 수 있는 프로그램 뼈대를 만드는 것입니다.

## 작동 확인표

- [ ] 핀 상수와 네오픽셀 객체를 직접 입력했다.
- [ ] `setupPins()`와 `setupNeoPixel()`을 작성했다.
- [ ] `setup()`에서 두 함수를 호출했다.
- [ ] 시험 색이 네오픽셀 8구에 표시되었다.
- [ ] RGB 값을 바꾸어 다른 색을 확인했다.
- [ ] 비어 있는 `loop()`가 반복 실행 영역임을 설명할 수 있다.

## 직접 바꾸어 확인하기

네오픽셀 시험 색을 파랑에서 주황으로 바꾸고 다시 업로드합니다. 어떤 숫자가 색을 결정하는지 설명한 뒤 시험 호출을 제거합니다. 다음 단계에서도 사용할 핀 상수와 초기화 함수는 남겨 둡니다.

## 오류 해결

| 증상 | 먼저 확인할 것 |
|---|---|
| 업로드가 되지 않음 | 보드와 포트 선택, USB 케이블 연결 |
| 글자가 깨짐 | 시리얼 모니터 속도를 115200으로 설정 |
| 네오픽셀이 켜지지 않음 | D7 연결, 5V와 GND 방향, 라이브러리 설치 |
| 거리값이 계속 측정 실패 | D2·D3 연결과 TRIG·ECHO 방향 |
| 센서값이 변하지 않음 | 해당 센서의 전원·GND·신호 핀 |
