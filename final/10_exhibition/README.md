# 10단계 최종 통합 실행본

5~9단계의 실제 기능을 한 화면에 합친 최종 페이지입니다.

- Arduino 센서 JSON 실시간 수신과 그래프
- 웹캠 MediaPipe 얼굴 특징 추출
- 사용자가 직접 모은 표정 샘플의 KNN 분류
- 거리·신뢰도·가변저항·조도에 따른 자동 판단
- 웹→Arduino 네오픽셀 명령
- Arduino `light_state` 응답에 따른 디지털 트윈 동기화
- 터치센서 상승 에지의 AUTO/MANUAL 전환

## 실행

이 폴더는 저장소의 다른 폴더 없이 단독으로 실행할 수 있습니다.

먼저 Arduino에 이 폴더 안의 다음 코드를 업로드합니다.

```text
arduino/10_final_integration/10_final_integration.ino
```

압축을 푼 이 폴더에서 명령 프롬프트를 열고 서버를 실행합니다.

```bash
py -m http.server 8000
```

`py`가 작동하지 않으면 Windows에서는 `python -m http.server 8000`, macOS에서는
`python3 -m http.server 8000`을 사용합니다.

Chrome 또는 Edge에서 다음 주소를 엽니다.

```text
http://localhost:8000/
```

`index.html`을 더블클릭해서 열면 모듈과 Web Serial이 정상 작동하지 않습니다.
Arduino IDE의 시리얼 모니터를 닫은 뒤 `Arduino 연결`을 누르세요.
AI 모델과 웹캠 기능은 MediaPipe 파일을 내려받으므로 인터넷 연결이 필요합니다.

## 시작 순서

1. `Arduino 연결`을 누르고 UNO 포트를 선택합니다.
2. 네 센서값이 들어오는지 확인합니다.
3. `AI 모델 준비`와 `웹캠 시작`을 차례로 누릅니다.
4. 각 표정을 3초씩 학습하고 `분류 시작`을 누릅니다.
5. `AUTO`를 선택하거나 터치센서를 한 번 누릅니다.
6. 실제 네오픽셀과 디지털 트윈이 같은지 확인합니다.

빠른 통신 시험은 `예시 데이터 사용`으로도 가능하지만, 전시 전에는 실제 사용자의 표정 샘플을 직접 학습하는 편이 안정적입니다.

## 자동 판단 우선순위

1. MANUAL이면 자동 명령을 만들지 않습니다.
2. 거리 측정 실패 또는 150cm 초과이면 조명을 끕니다.
3. 안정된 표정이나 70% 이상의 신뢰도가 없으면 이전 조명을 유지합니다.
4. 표정별 색을 선택합니다.
5. 가변저항으로 최대 밝기를 정합니다.
6. 실측된 조도 방향에 따라 최종 밝기를 보정합니다.

조도센서는 실측 결과에 따라 값이 클수록 어두운 `lower_is_brighter` 조건을 사용합니다.
