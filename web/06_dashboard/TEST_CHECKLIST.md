# 6단계 테스트 체크리스트

## 1. 자동 검사

- [ ] `node tests/parser.test.js` 실행 결과가 `모든 파서 테스트 통과`로 끝난다.
- [ ] `node --check data-parser.js`가 오류 없이 끝난다.
- [ ] `node --check app.js`가 오류 없이 끝난다.

## 2. 가상 센서 모드

- [ ] 페이지를 열면 별도 조작 없이 가상 센서가 실행된다.
- [ ] 네 카드의 숫자와 그래프가 약 0.2초 간격으로 바뀐다.
- [ ] `값을 자동으로 변화시키기`를 끄면 슬라이더로 값을 직접 조절할 수 있다.
- [ ] 터치 스위치를 켜면 카드가 `감지`로 바뀐다.
- [ ] `기록 지우기`를 누르면 그래프와 정상·오류 횟수가 0으로 초기화된다.
- [ ] 가상 모드를 끄면 값 갱신이 멈추고 상태 표시가 바뀐다.
- [ ] 가상 모드를 다시 켜면 값 갱신이 다시 시작된다.

## 3. 화면과 접근성

- [ ] 1440px 화면에서 센서 카드 4개가 한 줄로 보인다.
- [ ] 태블릿 너비에서는 카드가 2열로 보인다.
- [ ] 520px 이하에서는 카드와 조작부가 1열로 보이며 가로 스크롤이 생기지 않는다.
- [ ] `Tab` 키만으로 연결, 가상 모드, 슬라이더, 기록 지우기를 조작할 수 있다.
- [ ] 포커스가 있는 버튼과 입력 요소의 흰색 윤곽선이 선명하게 보인다.
- [ ] 브라우저 확대 200%에서도 버튼이나 값이 겹치지 않는다.
- [ ] 그래프마다 스크린리더용 최솟값·최댓값 설명이 갱신된다.

## 4. Web Serial 연결

Chrome 또는 Edge의 `https://` 주소나 `http://localhost`에서 확인합니다.

- [ ] Arduino를 연결하고 `Arduino 연결`을 누르면 포트 선택창이 열린다.
- [ ] 연결 후 버튼이 `연결 해제`로 바뀌고 가상 모드가 꺼진다.
- [ ] 속도 `115200 baud`로 JSON 한 줄이 들어올 때 카드와 그래프가 바뀐다.
- [ ] 1.6초 이상 데이터가 없으면 `데이터 지연` 상태가 표시된다.
- [ ] 다시 데이터가 오면 `데이터 수신 중`으로 복구된다.
- [ ] 연결 해제 후 가상 센서가 자동으로 다시 실행된다.
- [ ] USB 케이블을 갑자기 뽑아도 페이지가 멈추지 않고 가상 센서로 복구된다.

## 5. 데이터 형식과 오류 복구

아래 줄을 각각 보내 확인합니다.

| 입력 | 기대 결과 |
|---|---|
| `{"type":"sensor","pot":512,"light":730,"touch":false,"distance":38.4,"brightness":40,"color":1}` | 네 센서값 정상 표시, 조명 필드는 무시 |
| `{"type":"status","message":"pong"}` | 정상 Arduino 응답으로 안내하고 해석 오류는 늘리지 않음 |
| `512,730,0,38.4` | 네 값 정상 표시 |
| `가변저항: 512 \| 조도: 730 \| 터치: 감지 \| 거리(cm): 38.4` | 네 값 정상 표시 |
| `{"pot":512,"light":730,"touch":false,"distance":-1}` | 거리를 `실패`로 표시 |
| `{잘못된 JSON}` | 해석 오류가 1 증가하고 다음 데이터는 계속 처리 |
| `{"pot":2048,"light":730,"touch":false,"distance":38.4}` | 가변저항 경고가 표시되고 다른 정상값은 반영 |
| 빈 줄 | 오류를 늘리지 않고 무시 |
