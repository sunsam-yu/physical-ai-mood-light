# 6단계. 실시간 센서 대시보드

가변저항, 조도, 터치, 거리 값을 숫자 카드와 짧은 시계열 그래프로 보여 주는 정적 웹앱입니다. Arduino가 없어도 가상 센서가 기본으로 실행되므로 화면과 데이터 시각화를 먼저 실습할 수 있습니다.

## 실행

### 가장 간단한 방법

`index.html`을 브라우저에서 엽니다. 가상 센서 모드는 별도의 서버 없이 동작합니다.

### Web Serial까지 시험하는 방법

Web Serial은 보안 연결에서만 동작합니다. GitHub Pages에 배포하거나 이 폴더에서 로컬 서버를 실행한 뒤 Chrome 또는 Edge로 접속합니다.

```bash
python3 -m http.server 8000
```

그런 다음 `http://localhost:8000`을 열고 **Arduino 연결**을 누릅니다. 통신 속도는 `115200 baud`입니다.

## 화면 기능

- 4개 센서의 현재값, 단위, 의미를 각각 카드로 표시
- 최근 12초 정도의 변화를 Canvas 그래프로 표시
- 가상 센서 자동 변화 또는 슬라이더 수동 조절
- Web Serial 연결·해제 및 연결 상태 표시
- 정상 데이터, 해석 오류, 마지막 데이터 시각 표시
- 잘못된 한 줄이 들어와도 대시보드를 멈추지 않고 다음 줄 계속 처리
- 작은 화면, 키보드 탐색, 고대비 설정, 모션 감소 설정 지원

## 기본 데이터 형식

Arduino는 한 번 측정할 때마다 아래 JSON을 한 줄로 보내는 것을 권장합니다.

```json
{"type":"sensor","pot":512,"light":730,"touch":false,"distance":38.4,"brightness":40,"color":1}
```

각 줄의 끝에는 줄바꿈(`\n`)이 있어야 합니다. `distance`가 `-1` 또는 `null`이면 초음파 측정 실패로 표시합니다.

앞 단계 코드와의 연결을 쉽게 하기 위해 아래 형식도 읽을 수 있습니다.

```text
512,730,0,38.4
pot=512, light=730, touch=0, distance=38.4
가변저항: 512 | 조도: 730 | 터치: 없음 | 거리(cm): 38.4
```

6단계는 위 데이터에서 `pot`, `light`, `touch`, `distance`를 사용하고 조명 상태인 `brightness`, `color`는 다음 단계를 위해 그대로 무시합니다. `type: "status"` 응답과 `type: "error"` 오류 메시지도 구분합니다. 파서는 일부 영문 별칭과 한글 키도 지원하지만, 최종 통합에서는 권장 키를 사용하는 편이 오류를 줄입니다.

## 파일 구성

| 파일 | 역할 |
|---|---|
| `index.html` | 대시보드 구조와 접근성 정보 |
| `styles.css` | 반응형·전시용 화면 스타일 |
| `data-parser.js` | JSON·CSV·키-값 데이터 해석과 범위 검사 |
| `app.js` | Web Serial, 가상 센서, 카드, 그래프 제어 |
| `tests/parser.test.js` | 데이터 파서 자동 검사 |
| `TEST_CHECKLIST.md` | 브라우저·실물 보드 확인 목록 |

## 자동 검사

Node.js가 있으면 별도 패키지 설치 없이 파서 검사를 실행할 수 있습니다.

```bash
node tests/parser.test.js
node --check app.js
```

## 키트 연결 뒤 확인할 항목

- 실제 5단계 Arduino 코드가 권장 JSON을 줄 단위로 전송하는지
- 조도센서의 값 방향과 실제 범위
- 터치센서가 눌림을 `true/1`로 보내는지
- 초음파 측정 실패를 `-1` 또는 `null`로 보내는지
- 200ms 전송 간격에서 카드와 그래프가 안정적으로 갱신되는지

브라우저 수동 검증은 [`TEST_CHECKLIST.md`](TEST_CHECKLIST.md)를 따릅니다.
