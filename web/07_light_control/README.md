# 7단계. 웹에서 무드등 제어

웹에서 RGB·밝기·AUTO/MANUAL 모드를 정해 조명 명령을 보냅니다. 같은 명령을 8구 네오픽셀 2D 디지털 트윈에 표시하므로 Arduino가 없어도 `Mock` 전송으로 전체 흐름을 연습할 수 있습니다.

## 실행 방법

별도의 설치나 빌드 과정은 없습니다. 이 폴더를 로컬 웹 서버로 실행합니다.

```bash
python3 -m http.server 8000
```

브라우저에서 `http://localhost:8000/web/07_light_control/`을 엽니다.

- 키트 없이 확인: `가상 연결(Mock)` → `연결`
- 실제 Arduino 확인: `Arduino USB(Web Serial)` → `연결` → 장치 선택
- Web Serial은 Chrome 또는 Edge의 `https://` 또는 `localhost`에서 사용합니다.
- Arduino IDE의 시리얼 모니터가 열려 있으면 포트를 사용할 수 없으므로 먼저 닫습니다.

## 학생 실습 순서

1. `가상 연결(Mock)`을 연결합니다.
2. MANUAL 모드에서 색과 밝기를 정합니다.
3. `조명 명령 보내기`를 누릅니다.
4. 8개 픽셀과 최근 전송 데이터가 같은 값을 나타내는지 확인합니다.
5. RGB 입력에 범위를 벗어난 값을 넣어 오류 처리를 확인합니다.
6. 밝기를 80보다 크게 입력해 안전 상한으로 보정되는지 확인합니다.
7. AUTO 모드로 바꾸고 자동 제어에 넘기는 명령을 보냅니다.

## 명령 어댑터

화면은 직렬 포트에 직접 데이터를 쓰지 않고 아래 인터페이스만 사용합니다.

```js
await window.lightCommandAdapter.sendCommand({
  type: "light",
  r: 59,
  g: 130,
  b: 246,
  brightness: 40,
  mode: "MANUAL",
});
```

`LightCommandAdapter`가 입력 검증, 직렬화, 실제/가상 전송을 담당합니다. 5단계의 최종 통신 규격이 달라져도 `app.js`의 `serializeLightCommand()`만 수정하면 UI와 디지털 트윈은 그대로 사용할 수 있습니다.

## 기대하는 Arduino 명령 프로토콜

웹→Arduino 조명 명령은 UNO에서도 짧고 쉽게 해석할 수 있도록 UTF-8 CSV 한 줄로 보냅니다. 명령 끝에는 줄바꿈 문자(`\n`)를 붙입니다.

```text
LIGHT,59,130,246,40,MANUAL
```

| 순서 | 필드 | 허용값 | 역할 |
|---|---|---|---|
| 1 | 명령 이름 | `LIGHT` | 조명 명령 구분 |
| 2~4 | R, G, B | `0~255` 정수 | 네오픽셀 RGB |
| 5 | 밝기 | `0~80` 정수 | USB 전원을 고려한 밝기 상한 |
| 6 | 모드 | `MANUAL`, `AUTO` | 웹 수동 제어 또는 자동 제어 |

Arduino는 쉼표로 필드를 나눈 뒤 필드 수와 모든 범위를 검증하고, 올바른 명령일 때만 조명 상태를 한 번에 바꾸는 것을 권장합니다. AUTO 모드에서도 RGB와 밝기를 함께 보내며, Arduino는 이를 전환 직전 상태 또는 자동 제어 실패 시의 대체값으로 사용할 수 있습니다.

> 5단계 Arduino 완성 코드는 통신 원리를 익히기 위해 `PING`, `GET`만 처리합니다. 실제 조명 제어에는 [`arduino/07_light_control`](../../arduino/07_light_control)의 코드를 업로드합니다. 이 코드는 고정 길이 버퍼와 범위 검증 파서를 사용합니다.

Arduino→웹 센서 데이터는 5단계와 같은 JSON Lines를 계속 사용합니다. 즉, 방향에 따라 형식이 다릅니다.

- 웹→Arduino: `LIGHT,59,130,246,40,MANUAL`
- Arduino→웹: `{"type":"sensor", ...}`

## 안전·오류 처리

- RGB는 `0~255` 정수만 허용하며 잘못된 값은 전송하지 않습니다.
- 밝기는 앞 단계와 같은 `0~80`으로 제한합니다.
- AUTO 모드에서는 수동 RGB·밝기 조작을 잠가 현재 제어 주체를 분명히 표시합니다.
- 연결되지 않은 상태에서는 전송 버튼을 비활성화합니다.
- USB 분리나 쓰기 실패가 생기면 연결 상태를 해제하고 재연결을 안내합니다.
- 디지털 트윈은 마지막 명령 또는 현재 미리보기를 나타냅니다. 실제 Arduino 적용 여부를 확인하는 상태 응답은 9단계에서 추가합니다.

## 파일 구성

| 파일 | 역할 |
|---|---|
| `index.html` | 연결·조명 제어·디지털 트윈 화면 |
| `style.css` | 반응형 레이아웃과 8구 픽셀 표현 |
| `app.js` | 입력 검증, 명령 어댑터, Mock/Web Serial 전송 |

## 도전과제

1. `일몰`, `집중`, `휴식` 프리셋을 새로 추가합니다.
2. 8개 픽셀에 서로 다른 색을 보내는 `pixels` 배열 명령을 설계합니다.
3. Arduino가 `{"type":"light_state", ...}` 응답을 보내도록 만들고, 명령이 실제 적용된 뒤에만 디지털 트윈을 갱신합니다.
4. 연결이 끊겼다가 다시 연결되면 마지막 MANUAL 명령을 사용자의 확인 후 다시 보내도록 만듭니다.
5. AUTO 모드에서 조도·거리값에 따라 조명 밝기가 달라지는 규칙을 순서도로 설계합니다.
