# 실물 키트 검증 순서

키트를 가져온 뒤 아래 순서대로 진행합니다. 처음부터 모든 부품을 연결하지 않습니다.

## 0. 준비

- Arduino UNO, USB 데이터 케이블
- 가변저항, 조도센서, 터치센서, 초음파센서, 네오픽셀 8구
- 점퍼선과 브레드보드
- Arduino IDE, Chrome 또는 Edge

전원을 넣기 전 각 모듈의 `5V·GND·SIG` 인쇄 방향을 사진으로 확인합니다.

## 1. 보드와 네오픽셀

1. UNO 포트와 보드 종류를 선택합니다.
2. 3단계 네오픽셀 예제를 업로드합니다.
3. 8개 LED의 색 순서와 밝기 80에서 전원 안정성을 확인합니다.

## 2. 센서 개별 측정

| 부품 | 시험 조건 | 기록할 값 |
|---|---|---|
| 가변저항 | 최소·중간·최대 |  |
| 조도센서 | 가림·실내·휴대전화 손전등 |  |
| 터치센서 | 손 뗌·접촉 | HIGH/LOW: |
| 초음파 | 10·30·100cm |  |

조도센서는 밝을수록 값이 커지는지 작아지는지를 이 표에서 확정합니다.

## 3. 모듈형 통합

4단계 코드를 업로드하고 센서 네 종류가 동시에 갱신되는지 확인합니다. 터치를 눌러도 거리 측정과 시리얼 출력이 멈추지 않아야 합니다.

## 4. 웹 연결과 제어

1. 5단계 코드와 웹에서 JSON 센서값 수신을 확인합니다.
2. 6단계 카드와 그래프가 실제 값에 반응하는지 확인합니다.
3. USB를 분리해 끊김 표시와 재연결 절차를 시험합니다.
4. 7단계 Arduino 코드를 올리고 RGB·밝기·모드를 전송합니다.

## 5. 웹캠 AI와 자동 조명

1. 8단계에서 모델 준비 → 웹캠 → 네 범주 학습 → 분류를 시험합니다.
2. 범주별 15개와 30개 샘플의 차이를 기록합니다.
3. 개별 측정 결과로 9단계 기준을 확정합니다.

| 설정 | 임시값 | 실측 후 확정 |
|---|---:|---:|
| 사람 감지 최대 거리 | 150cm |  |
| AI 최소 신뢰도 | 70% |  |
| 네오픽셀 최대 밝기 | 80 |  |
| 조도 방향 | 미정 |  |
| 터치 활성 신호 | HIGH 가정 |  |

## 중단 조건

- 부품이나 점퍼선이 뜨거워짐
- UNO가 반복 재부팅됨
- 네오픽셀 일부가 비정상적으로 깜빡임
- 센서 전원 핀 배열이 자료와 다름

이 경우 즉시 USB를 분리하고 결선 사진과 모듈명을 확인합니다.
